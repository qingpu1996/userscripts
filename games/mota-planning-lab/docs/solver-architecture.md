# 魔塔求解器方向

状态：Rust shadow runtime 尚未实现。

本仓库当前保留浏览器前端、最小协议 Schema 和 Rust Stage 0 spike。旧 Python 后端及其运行、恢复、规划和存储实现已移除，因此不存在可启动的 solver runtime。

## 目标运行边界

```text
JS 实时采集当前游戏状态
  → Rust 在进程内存中计算一个建议
  → JS 最多执行一个动作
  → 下一轮重新采集实时状态
```

实时 observation 是运行事实来源。Rust 的索引、搜索缓存和当前候选只存在于本次进程内存；旧 action、世界和规划状态不持久化，也不恢复或重放。唯一允许的持久化是本次启动的旁路全量日志，供事后复盘，运行循环只写不读。

## 下一步

1. 实现只读 Rust shadow runtime：接收 JS 实时状态并返回建议，不执行页面动作。
2. 用固定 fixture 和最小协议契约核对 JS/Rust 字段、规范化及陈旧响应处理。
3. 在 shadow 通过独立验收后，再评估页面侧单步执行接入。

不得把 Stage 0 的合成结果当作策略正确性、服务 SLA 或真实存档自动驾驶授权。
