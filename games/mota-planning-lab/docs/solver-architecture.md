# 魔塔求解器方向

状态：Stage3 已实现受支持规则子集的全局只读路线证明；真实页面执行未实现。

浏览器把明确终局、资源增量、门、普通怪物、受限商店和精确换层投影为结构化事实。Rust 在一次 observation 上跨楼层搜索完整终局路线，以通关为硬约束；完整路线依次最大化终局攻防总和、攻防较小值和终局 HP，最后用路线序列确定性破同分。金币、经验、钥匙和背包不参与终局评分。输出仍为 `idle + shadow` 的只读路线和首步建议，不包含可执行字段。

## 目标运行边界

```text
JS 实时采集当前游戏状态
  → Rust 在进程内存中计算一个建议
  → JS 只展示全局证明与只读建议
  → 下一轮重新采集实时状态
```

实时 observation 是唯一运行事实来源。Rust 只保留进程内 shadow cycle；状态搜索有界，预算耗尽返回 `unproven`。动态脚本、未知怪物 special、无法确定落点或终局的事件返回 `unsupported` blocker，绝不退回局部贪心。

## 下一步

1. [完成] 实现只读 Rust shadow runtime：接收 JS 实时状态并返回建议，不执行页面动作。
2. [完成] 用固定 fixture 和真实 HTTP 核对 JS/Rust 字段与 `idle + shadow` 合同。
3. [完成] 在真实页面完成只读联通验证，并增加当前楼层边界与即时成本分析。
4. [完成] 对结构化支持子集实现跨层终局证明、终局攻防优先且 HP 次优的字典序优化和确定性首步建议。
5. [待验证] 用真实页面 observation 检查终局、资源和换层投影覆盖率；仍不执行建议。

不得把 Stage 0 的合成结果当作策略正确性、服务 SLA 或真实存档自动驾驶授权。
